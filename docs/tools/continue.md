# Continue

## 工具定位

- 定位：AI checks / PR 审查（开源 CLI + 平台）
- 入口：`cn` CLI + GitHub 状态检查 + CI

## 功能列表

```{list-table}
:header-rows: 1

* - 能力
  - 状态
  - 说明
* - Checks as Code
  - ✅
  - 规则可版本化维护
* - PR 通过/失败门禁
  - ✅
  - 可作为 CI 检查
* - 建议修复
  - ✅
  - 对审查结果给出建议
* - 本地/CI 运行
  - ✅
  - 便于团队落地
```

## 适合场景

- 想把 AI 审查“流程化、可执行化”的团队

## 主要限制/注意点

- 需要先建设 checks 规则，初期有配置成本

## 详细评价

### 优势

- 把 AI 审查流程化。

### 局限

- 初期要建设并维护 checks 规则。
